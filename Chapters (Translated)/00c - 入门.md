# 入门

## 设置您的环境

1. 安装 F#。最简单的方法是安装最新的 [.NET SDK](<https://dotnet.microsoft.com/download>)（发布时为 6.0.x）。
2. 安装 Visual Studio Code (VS Code)。
3. 为 VS Code 安装 ionide F# 扩展。您可能需要重新加载环境。扩展的安装页面会告诉您。
4. 为本书创建一个文件夹，然后为每一章创建文件夹以存放您将编写的代码。

### F# 交互式 REPL（FSI）

如果您从 C# 转到 F#，F# 交互式窗口类似于 C# 交互式窗口。主要的不同之处在于，F# 开发人员不依赖逐行调试，而是使用 FSI 来验证代码。在进行 F# 开发时，我从不使用断点调试代码。

FSI 出现在 VS Code 的终端窗口中。除了 .NET 终端，您还可以访问 F# 交互式窗口。您可以直接在 FSI 中输入代码，但我们将从文件中直接发送代码到 FSI 进行编译和运行。

> 通过选择要运行的代码并按 ALT+ENTER，我们可以在 FSI 中运行代码。代码将被复制到 FSI 中，编译并运行。如果您不更改该代码，FSI 会将编译后的代码保存在内存中以供以后重用。

如果您直接在 FSI 中输入代码，您需要以两个分号 ```;;``` 结束输入，以告诉 FSI 执行您的代码。

### 脚本文件与代码文件

F# 支持两种类型的文件：脚本 (.fsx) 和代码 (.fs)。在本书中我们将使用这两种类型的文件。它们在 VS Code 中的主要区别在于，只有 .fs 文件会被编译成输出的 .exe 或 .dll 文件，而 .fsx 文件主要用于在 F# 交互式中尝试新东西。

任何类型的 F# 代码文件都可以通过使用 ALT + ENTER 发送到 F# 交互式中运行。

FSI 还有一个命令行版本，可以用来运行脚本文件，但这超出了本书的范围。

### 了解 VS Code 和 Ionide

虽然 VS Code 和 ionide 环境不像完整的 IDE（Visual Studio 和 Jet Brains Rider）那样功能齐全，但它并不是没有功能。即使是处理大型 F# 代码库，您实际上也不需要太多功能。

[Compositional-IT](<https://www.compositional-it.com/>) 在 YouTube 上发布了一些视频，介绍了 [VS Code + F# 快捷键](<https://www.youtube.com/playlist?list=PLlzAi3ycg2x27lPUwp3Z-M2m44n681cED>)。

[Ionide](<https://ionide.io/index.html>) 远不止是 VS Code 的 F# 扩展。花些时间看看他们提供的内容，以帮助您更好地体验 F#。
