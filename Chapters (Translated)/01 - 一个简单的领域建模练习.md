## 旅途开始

除了简单的数据类型如字符串、十进制和布尔值，F# 还有一个强大的代数类型系统（ATS）。可以将这些类型视为简单的数据结构，用于构建更大的数据结构。我们使用 `type` 关键字来定义一个类型，如下所示的 *元组（Tuple）* 类型：

```fsharp
type Customer = string * bool * bool
```

> 译者注：代数数据类型（Algebraic data type）是一种复合类型，常用于函数式编程和类型论。常见的代数数据类型有**积**类型（元组和记录）和**和**类型（枚举）

这个定义表明，*Customer* 类型是一个包含一个 *string* 和两个 *booleans* 的元组。这意味着一个 *元组* 是定义为一个 *AND* 类型。F# 中的元组没有命名部分，所以使用你类型的用户必须猜测数据的含义。注意类型名称开头的大写字母，这通常被称为 *帕斯卡命名法*。除了类型定义之外，大多数情况使用 *小驼峰命名法*，即首字母小写。

我们使用 `let` 关键字创建一个 *Customer* 类型的实例：

```fsharp
// string * bool * bool
let fred = ("Fred", true, true)
```

这个例子中的括号是可选的。注意定义中使用了 `*`，而在使用时使用 `,` 来分隔元组的各部分。

使用 `let`，我们将数据 `("Fred", true, true)` 绑定到标识符 `fred`。默认情况下，F# 中所有数据都是不可变的，所以不应该将 `fred` 视为变量。

我们可以通过在我们的 *let 绑定* 中添加类型显式地声明 `fred` 标识符绑定的数据类型：

```fsharp
// Customer
let fred : Customer = ("Fred", true, true)
```

我们也可以使用 `let` 关键字来解构元组：

```fsharp
let (id, isEligible, isRegistered) = fred
```

我们将标识符 `id`、`isEligible` 和 `isRegistered` 分别绑定到 `fred` 绑定中不同位置的数据。

> 译者注：Eligible：有资格的

与其使用 *元组*，我们将创建一个 *记录（Record）* 类型来解决属性名称问题。我们可以这样定义初始的客户记录类型：

```fsharp
type Customer = { Id:string; IsEligible:bool; IsRegistered:bool }
```

记录类型和元组一样，是一个 *AND* 类型，因此在这个例子中，一个 *Customer* 由一个字符串值的 Id 和两个布尔值 *IsEligible* 和 *IsRegistered* 组成。记录类型，像我们将在本书中看到的大多数类型一样，是不可变的，即一旦创建就不能更改。这意味着创建 *Customer* 记录的所有数据必须在实例创建时提供。

我们也可以将每个字段放在单独的一行上，而不是在一行上定义记录类型。

> 警告：F# 不支持制表符，因此你的 IDE/编辑器需要能够将制表符转换为空格，大多数编辑器都支持这一功能，当然包括 VS Code。

```fsharp
type Customer = {
    Id : string
    IsEligible : bool
    IsRegistered : bool
}
```

注意属性是如何对齐的。F# 使用 *Significant Whitespace*，因此对齐对于作用域很重要。如果编译器警告你代码中有错误，对齐问题是你首先应该检查的内容之一。

如果你将命名值放在不同的行上，就不再需要使用分号分隔符。名称值标签和类型之间允许有空格，并且它们不需要一致。

要创建一个客户实例，我们可以在类型定义 **下面** 编写以下代码：

```fsharp
let fred = { Id = "Fred"; IsEligible = true; IsRegistered = true } // Customer
```

F# 编译器推断出绑定到 *fred* 标识符的值类型是 Customer。通过使用 let 关键字，我们将标识符 *fred* 绑定到了那个 Customer 实例。

我们还有另一种选择，这种方式在你有更多属性时更好，使你的代码更易读：

```fsharp
let fred = { 
    Id = "Fred"
    IsEligible = true
    IsRegistered = true 
}
```

注意我们不再需要使用分号来分隔属性定义。

如果你愿意，可以在值绑定上注释类型：

```fsharp
let fred : Customer = { Id = "Fred"; IsEligible = true; IsRegistered = true }
```

如果定义了多个具有相同结构的记录类型，编译器将无法推断类型，因此我们需要用类型注释标识符。

我们不会使用 *fred* 绑定，所以你可以删除那行代码。

如果你使用过 C# 或 Java 等语言，你可能会对代码的严格顺序感到惊讶。F# 编译器从文件顶部向下工作。起初这可能看起来有点奇怪，但你很快就会习惯。它对我们编写和验证代码有一些非常好的优点，同时也使编译器的工作更容易。这种顺序也适用于项目级别，所以你的源代码（*.fs*）文件需要以同样的方式排序，而不是按字母顺序排序。

在 *Customer* 类型下面，我们需要创建一个函数来计算总额，该函数接受一个 *Customer* 和一个 *Spend*（decimal）作为输入参数，并返回 *Total*（decimal）作为输出：

```fsharp
// Customer -> decimal -> decimal
let calculateTotal (customer:Customer) (spend:decimal) : decimal =
    let discount = 
        if customer.IsEligible && spend >= 100.0M 
        then (spend * 0.1M) else 0.0M   
    let total = spend - discount
    total
```

数字末尾的 M 后缀告诉编译器该数字是一个十进制数。

关于函数，有几点需要注意：

- 我们再次使用 ```let``` 来定义函数，并在函数内部定义折扣和总额绑定。
- 折扣和总额绑定的作用域仅限于函数内部，在函数外部不可见。
- 没有容器，例如类，因为函数是一级公民。
- 返回类型位于输入参数的右侧。
- 不需要 return 关键字，因为最后一行会自动返回。
- 使用显著的空白来定义作用域，嵌套时不允许使用制表符。
- 我们使用了一个 ```if``` 表达式，它必须为 true 和 false 路径返回相同类型。

> **表达式** 在 F# 编程中被广泛使用，因为它们总是返回一个输出。这使得它们易于与其他表达式组合并易于测试。相比之下，语句不返回输出，因此不可组合。

在函数定义上方的注释显示了函数签名，即 **Customer -> decimal -> decimal**。签名末尾最后一个箭头后的项是函数的返回类型。这个函数签名的意思是 *这个函数接受一个 Customer 和一个 decimal 作为输入，并返回一个 decimal 作为输出*。这并不完全准确，你将在下一章中发现。你的 IDE 通常会显示函数签名，或者你可以通过将鼠标悬停在函数名称上查看它。

> **函数签名** 非常重要，正如你将在下一章中发现的那样；多留意它们。

*calculateTotal* 函数的输入参数是柯里化形式。柯里化参数意味着有多个参数。如果一个函数签名有多个箭头，那么你就有柯里化参数。我们也可以将它们安排成元组形式：

```fsharp
// (Customer * decimal) -> decimal
let calculateTotal (customer:Customer, spend:decimal) : decimal =
    let discount = 
        if customer.IsEligible && spend >= 100.0M 
        then (spend * 0.1M) else 0.0M   
    let total = spend - discount
    total
```

注意函数签名从 **Customer -> decimal -> decimal** 变为 **(Customer * decimal) -> decimal**。函数主体保持不变，只是输入参数变成了单一的元组参数。使用柯里化形式对大多数 F# 函数有显著的好处，所以这是你在本书中主要看到的风格。我们将在下一章中发现一些好处以及名称的由来。

F# 编译器支持一个称为 **类型推断** 的特性，这意味着大多数情况下它可以通过使用来确定类型，而不需要你显式地定义它们。因此，我们可以将函数重写为：

```fsharp
// Customer -> decimal -> decimal
let calculateTotal customer spend =
    let discount = 
        if customer.IsEligible && spend >= 100.0M then spend * 0.1M 
        else 0.0M   
    spend - discount
```

IDE 仍然应该显示函数签名为 **Customer -> decimal -> decimal**。我还删除了 total 绑定，因为我认为它对函数的可读性没有任何帮助。

> 不要忘记选中你到目前为止编写的代码，然后按 ALT + ENTER 在 F# 交互式 (FSI) 中运行它。

虽然类型推断工作得非常好，但在某些情况下它不起作用，你将需要为参数提供类型。其中一个情况是你使用 .NET 函数如 ```DateTime.TryParse```。问题在于该函数有两个重载，编译器无法确定需要使用哪个。

## 将隐式变为显式

创建一个名为 *part2.fsx* 的新文件，并将 part1.fsx 中的最终代码复制到其中。在本节中，我们将修改 part2.fsx 中的代码。

首先，我们为 *Registered* 和 *Unregistered* 客户创建特定的记录类型。

```fsharp
type RegisteredCustomer = {
    Id : string
    IsEligible : bool
}

type UnregisteredCustomer = {
 Id : string
}
```

为了表示客户可以是 *Registered* 或 *Unregistered*，我们将使用代数类型系统中的另一种内置类型：判别联合 (DU)。我们像这样定义 Customer 类型：

```fsharp
type Customer =
    | Registered of RegisteredCustomer
    | Guest of UnregisteredCustomer
```

这表示“客户可以是类型为 *RegisteredCustomer* 的 *Registered* 或类型为 *UnregisteredCustomer* 的 *Guest*”。这些项称为联合案例，由一个案例标识符（在本例中为 *Registered*/*Guest*）和一些可选的案例数据组成。你可以选择性地将任何类型或类型的混合附加到一个联合案例中。

判别联合是封闭集合，因此只有类型定义中描述的案例是可用的。唯一可以定义案例的地方是在类型定义中。

> 译者注：[Microsoft Learn 上的可区分联合](https://learn.microsoft.com/dotnet/fsharp/language-reference/discriminated-unions)

理解判别联合的最简单方法是使用它们！我们必须对已定义的用户进行更改。首先是 *UnregisteredCustomer*：

```fsharp
// Guest of UnregisteredCustomer
let sarah = Guest { Id = "Sarah" } // Customer
```

在这个例子中，*sarah* 仍然是一个 Customer，但你不能创建一个 Customer，你必须是其中一个案例：Registered 或 Guest。

看看判别联合中的定义与案例定义的比较。

现在让我们对 *RegisteredCustomer* 绑定进行必要的更改：

```fsharp
// Registered of RegisteredCustomer
let john = Registered { Id = "John"; IsEligible = true }
let mary = Registered { Id = "Mary"; IsEligible = true }
let richard = Registered { Id = "Richard"; IsEligible = false }
```

将 *Customer* 类型从记录更改为判别联合也会影响 *calculateTotal* 函数。我们将使用另一个 F# 特性来修改折扣计算：使用 match 表达式进行模式匹配：

```fsharp
let calculateTotal customer spend =
    let discount = 
        match customer with
        | Registered c -> 
            if c.IsEligible && spend >= 100.0M then spend * 0.1M else 0.0M
        | Guest _ -> 0.0M
    spend - discount
```

我们使用 match 表达式来确定 *Customer* 的案例。

> **表达式** 产生一个输出值。在 F# 中几乎所有东西，包括值、函数和控制流，都是表达式。这使它们易于组合。

要理解模式匹配在做什么，请比较 match ```Registered c``` 与我们构造用户的方式：```Registered { Id = "John"; IsEligible = true }```。在这种情况下，```c``` 是 *RegisteredCustomer* 案例数据的占位符。在 *Guest* 模式匹配中的下划线是一个通配符，表示我们不需要访问该案例数据。对判别联合进行模式匹配是全面的；也就是说，每个案例都必须由 match 表达式处理。如果你不处理每个案例，编译器会发出警告，提示“模式匹配不完整”。

我们可以通过添加守卫子句来简化逻辑，但这意味着我们需要考虑不符合条件的注册客户，否则匹配是不完整的：

```fsharp
let calculateTotal customer spend =
    let discount = 
        match customer with
        | Registered c when c.IsEligible && spend >= 100.0M -> spend * 0.1M
        | Registered _ -> 0.0M
        | Guest _ -> 0.0M
    spend - discount
```

我们可以使用通配符来简化最后两个匹配，如下所示：

```fsharp
let calculateTotal customer spend =
    let discount = 
        match customer with
        | Registered c when c.IsEligible && spend >= 100.0M -> spend * 0.1M
        | _ -> 0.0M
    spend - discount
```

> 使用通配符时要小心，因为它可能会阻止编译器警告你对判别联合的添加，并可能导致意外结果。

我认为这比我们开始时的版本要好得多，因为可读性得到了提高，并且我们防止了进入无效状态。然而，通过用领域概念替换原始类型，我们仍然可以做得更好；我们将在后面的章节中重新讨论这一点。另一个改进是添加 XUnit 单元测试，在那里我们可以利用已经编写的辅助函数和断言。

现在 *RegisteredCustomer* 和 *UnregisteredCustomer* 记录真的很简单，我们可以删除它们，并将字符串 Id 值直接添加到 Customer 判别联合中的案例值：

```fsharp
type Customer =
    | Eligible of Id:string
    | Registered of Id:string
    | Guest of Id:string

let calculateTotal customer spend =
    let discount = 
        match customer with
        | Eligible _ when spend >= 100.0M -> spend * 0.1M
        | _ -> 0.0M
    spend - discount

let john = Eligible "John"
let mary = Eligible "Mary"
let richard = Registered "Richard"
let sarah = Guest "Sarah"

let assertJohn = calculateTotal john 100.0M = 90.0M
let assertMary = calculateTotal mary 99.0M = 99.0M
let assertRichard = calculateTotal richard 100.0M = 100.0M
let assertSarah = calculateTotal sarah 100.0M = 100.0M
```

我选择不这样做，因为 *RegisteredCustomer* 案例数据可能包含比我们目前看到的更多的数据。这并不妨碍你只对 *UnregisteredCustomer* 类型记录进行此操作。

这是一种对这个简单领域进行建模的方法，但我们也可以用其他几种风格来做。

## 替代方法 (1 of 2)

创建一个名为 *part4.fsx* 的新文件，并将 part3.fsx 的最终代码复制到其中。在本节中，我们将修改 *part4.fsx* 中的代码。

另一种方法是使用判别联合作为记录类型属性中的一种类型：

```fsharp
type CustomerType =
    | Registered of IsEligible:bool
    | Guest

type Customer = { Id:string; Type:CustomerType }

// Customer -> decimal -> decimal
let calculateTotal customer spend =
    let discount =
        match customer.Type with
        | Registered (IsEligible = isEligible) when isEligible && spend >= 100.0M -> spend * 0.1M
        | _ -> 0.0M
    spend - discount

let john = { Id = "John"; Type = Registered (IsEligible = true) }
let mary = { Id = "Mary"; Type = Registered (IsEligible = true) }
let richard = { Id = "Richard"; Type = Registered (IsEligible = false) }
let sarah = { Id = "Sarah"; Type = Guest }

let assertJohn = calculateTotal john 100.0M = 90.0M
let assertMary= calculateTotal mary 99.0M=99.0M 
let assertRichard=calculateTotal richard 100.0M=100.0M 
let assertSarah=calculateTotal sarah 100.0M=100.0M 
```

这是完全有效的，但这意味着引入了当前领域理解中没有的语言。

我们可以像这样简化过滤器：

```fsharp
let calculateTotal customer spend =
    let discount =
        match customer.Type with
        | Registered (IsEligible=true) when spend >= 100.0M -> spend * 0.1M 
        | _ -> 0.0M 
    spend - discount 
```

它很好地解决了非法状态问题，但我不喜欢不得不发明名字。我更喜欢使我的代码尽可能地以领域为中心。

## 替代方法 (2 of 2)

还记得我们之前不推荐使用元组，因为它们不允许命名各个部分吗？判别联合有自己的数据结构可以解决这个问题。尽管它们看起来像元组，但判别联合的数据结构不是元组。

创建一个名为 *part5.fsx* 的新文件，并将 part2.fsx 的最终代码复制到其中。在本节中，我们将修改 *part5.fsx* 中的代码。

让我们从修改 *Customer* 类型和 *calculateTotal* 函数开始：

```fsharp
type Customer =
    | Registered of Id:string * IsEligible:bool
    | Guest of Id:string

// Customer -> decimal -> decimal
let calculateTotal customer spend =
    let discount =
        match customer with
        | Registered (id, isEligible) when isEligible && spend >= 100.0M -> spend * 0.1M
        | _ -> 0.0M
    spend - discount
```

你不需要修改断言，但我们确实需要更改 let 绑定以支持新的 Customer 数据结构：

```fsharp
let john = Registered (Id = "John", IsEligible = true)
let mary = Registered (Id = "Mary", IsEligible = true)
let richard = Registered (Id = "Richard", IsEligible = false)
let sarah = Guest (Id = "Sarah")
```

高亮显示此脚本文件中的所有代码，并在 FSI 中运行以验证所有断言都返回 true。

如果我们不需要 Id 属性的值，我们可以使用通配符忽略它：

```fsharp
// Customer -> decimal -> decimal
let calculateTotal customer spend =
    let discount =
        match customer with
        | Registered (_, isEligible) when isEligible && spend >= 100.0M -> spend * 0.1M
        | _ -> 0.0M
    spend - discount
```

我们还可以像之前的版本那样简化过滤器：

```fsharp
// Customer -> decimal -> decimal
let calculateTotal customer spend =
    let discount =
        match customer with
        | Registered (IsEligible = true) when spend >= 100.0M -> spend * 0.1M
        | _ -> 0.0M
    spend - discount
```

如果你需要 Id 属性的值，我们可以在应用 `IsEligible = true` 过滤器的同时请求它：

```fsharp
// Customer -> decimal -> decimal
let calculateTotal customer spend =
    let discount =
        match customer with
        | Registered (Id = id; IsEligible = true) when spend >= 100.0M -> spend * 0.1M
        | _ -> 0.0M
    spend - discount
```

如果你在查看我们编写的一些过滤器并想知道它们是否可以提取为某种可重用的代码，那么你很幸运！你正在寻找的特性叫做活动模式。我们将在本书后面的章节中专门讨论它们，但这是一个在我们代码版本中使用的示例：

```fsharp
// Customer -> unit option
let (|IsEligible|_|) customer =
    match customer with
    | Registered (IsEligible = true) -> Some ()
    | _ -> None

// Customer -> decimal -> decimal
let calculateTotal customer spend =
    let discount =
        match customer with
        | IsEligible when spend >= 100.0M -> spend * 0.1M
        | _ -> 0.0M
    spend - discount
```

你将在第三章遇到 Option、Some、None 和 Unit，在第七章遇到活动模式。我只是想让你提前了解一下你将在本书后面遇到的一些好东西。

领域建模的指导原则是尽可能以领域为中心，同时 [使非法状态不可表示](https://fsharpforfunandprofit.com/posts/designing-with-types-making-illegal-states-unrepresentable/)。建模充其量是一项不精确的任务。尝试多个版本，看看是否有比你最初想到的更好的方法。

## 总结

在本章中，我们结合了多种类型来解决一个简单的业务问题。尽管 F# 代数类型系统中的类型（记录和元组）和（判别联合）结构简单，但它们可以组合在一起构建复杂的数据结构。更重要的是，它可以构建出极其富有表现力的结构，以建模几乎任何类型的领域。
